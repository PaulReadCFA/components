Quick reference

All components live in cfa-components.css. The modifier pattern is .cfa-[component]--[variant].

CFA Program boxes vs additional boxes: .cfa-callout matches the curriculum / DesignPLUS production menu (shaded, example, exercise, etc.). .cfa-box uses the same layout with brand palette tints (lime, aqua, mauve, cyan) for new or repurposed tags outside that menu.

Canvas deployment

📋 CSS → Admin › Themes › Custom CSS

Copy the entire contents of cfa-components.css and paste into the Custom CSS field in your Canvas theme.

📋 JS → Admin › Themes › Custom JavaScript

Copy the entire contents of cfa-components.js and paste into the Custom JavaScript field.

📄 HTML → Canvas page editor

  • Switch to the HTML source view in the RCE editor
  • Paste the relevant HTML snippet from the demo pages
  • For MathJax in equation boxes: Canvas pages must load MathJax (or KaTeX) globally—see how demo-library.html / demo-exploration.html include the CDN script and mathjax-demo-boot.js for tabs/expanders. Theme-level script is the usual place.
  • For pages with multiple .cfa-exploration blocks, ensure tab/panel IDs are unique — use a different prefix per block (e.g. eq2-tab-1 / eq2-panel-1)
  • Canvas may strip some attributes — test after saving. The components use only class, id, aria-*, role, data-*, and tabindex, all of which Canvas allows
